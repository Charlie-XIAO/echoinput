1. Think Before Coding: Don't assume. Don't hide confusion. If uncertain, ask. If a simpler approach exists, say so.
2. Simplicity First: No features beyond what was asked. No abstractions for single-use code. No "flexibility" that nobody requested. If 200 lines could be 50, rewrite it.
3. Surgical Changes: Touch only what you must. Don't "improve" adjacent code. Don't refactor things that aren't broken. Every changed line should trace directly to your request.
4. Goal-Driven Execution: Don't tell the agent what to do. Give it success criteria and let it loop until verified. Write the test first, then make it pass.
